# Project agent instructions

## Completion and publishing

For every task that authorizes changes to this project, once the requested behavior is
implemented, verified in the relevant environment, and the appropriate checks pass:

- Commit all intentional task changes before reporting the work as complete.
- Push the commit to the configured remote branch before reporting completion.
- Never leave verified work only in the local working tree unless the user explicitly asks
  not to commit or push it.
- Never include unrelated user changes in the commit. If the working tree is mixed and the
  intended scope is unclear, ask before staging.
- If committing or pushing is blocked, report the blocker explicitly and do not describe the
  work as fully complete.
